# Пользовательский словарь - Руководство

## Что это такое?

Пользовательский словарь позволяет добавить свои слова, которые **не должны** помечаться как ошибки. Это полезно для:

- Технических терминов и названий (PyTorch, EasyOCR)
- Названий брендов и продуктов (YouTube, GitHub, OpenCV)
- Специфических слов вашего проекта
- Сокращений и аббревиатур
- Имен собственных

## Формат файла

Файл `custom_dictionary.txt` имеет простой текстовый формат:

```bash
# Комментарий - строки начинающиеся с # игнорируются

# Каждое слово на отдельной строке
слово1
слово2
слово3

# Регистр не важен - все слова преобразуются в нижний регистр
AdaptON
adapton
ADAPTON
# Все три варианта выше будут работать одинаково

# Пустые строки игнорируются

технология
информационные
знакомиться
```

## Правила

1. **Одно слово на строку** - каждое слово должно быть на отдельной строке
2. **Комментарии** - строки начинающиеся с `#` игнорируются
3. **Регистр** - слова проверяются без учета регистра (все в lowercase)
4. **Пустые строки** - игнорируются
5. **Кодировка** - файл должен быть в кодировке UTF-8

## Расположение файла

По умолчанию программа ищет файл `custom_dictionary.txt`:

- **При запуске из Python:** в той же папке, где находятся `video_speller.py` и `gui.py`
- **При запуске EXE:** в той же папке, где находится `VideoSpellChecker.exe`

### Редактирование через GUI

В графическом интерфейсе можно открыть словарь через меню:

**Файл → Открыть пользовательский словарь...**

Файл откроется в системном редакторе. Если файла нет, он будет создан автоматически.

### Программное использование

Вы можете указать другой путь при создании объекта:

```python
from gui_widgets import VideoSpellCheckerWithLogging

# Использовать другой файл словаря
checker = VideoSpellCheckerWithLogging(
    output_dir="screenshots_video_errors",
    custom_dict_path="my_words.txt"
)
```

## Быстрый старт

### Через GUI (рекомендуется)

1. Запустите программу: `python gui.py`
2. Откройте меню **Файл → Открыть пользовательский словарь...**
3. Добавьте свои слова (по одному на строку)
4. Сохраните файл (Ctrl+S)
5. Перезапустите программу - слова из словаря не будут помечаться как ошибки

### Вручную

1. Откройте файл `custom_dictionary.txt` в текстовом редакторе
2. Добавьте свои слова (по одному на строку)
3. Сохраните файл в кодировке UTF-8
4. Запустите программу - слова из словаря не будут помечаться как ошибки

## Проверка работы

Чтобы убедиться, что словарь загружен правильно, посмотрите на вывод в логе программы при запуске:

```bash
⏳ Загрузка моделей OCR (это может занять время при первом запуске)...
✓ Модели OCR загружены
⏳ Загрузка словарей...
✓ Русский словарь загружен
✓ Английский словарь загружен
✓ Загружено слов из пользовательского словаря: 25  ← Вот эта строка
```

Если вы видите сообщение "ℹ Пользовательский словарь пуст" или количество слов равно 0, значит файл не найден, пуст или находится не в той папке.

## Советы

1. **Группируйте слова** - используйте комментарии для организации слов по категориям
2. **Регулярно обновляйте** - добавляйте новые слова по мере работы с программой
3. **Делитесь с командой** - если работаете в команде, держите файл в системе контроля версий (git)
4. **Не перебарщивайте** - добавляйте только действительно правильные слова, иначе можете пропустить реальные ошибки

## Отладка

Если слово из словаря все равно помечается как ошибка:

1. Проверьте, что слово записано правильно (без опечаток)
2. Убедитесь, что перед словом нет пробелов или специальных символов
3. Проверьте кодировку файла (должна быть UTF-8)
4. Перезапустите программу - словарь загружается при старте

## Примечания

- Словарь загружается **один раз** при старте программы
- Изменения в файле вступают в силу после перезапуска
- Слова из словаря имеют **приоритет** над основными словарями
- Смешанные слова (например, "PyTorch123") нужно добавлять без цифр: "PyTorch"
